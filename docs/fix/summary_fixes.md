# Tổng hợp các vấn đề đã được khắc phục

Tài liệu này tổng hợp lại toàn bộ các vấn đề (lỗi bảo mật, lỗi code linter, lỗi testing) đã được phát hiện và cách khắc phục trong dự án.

## Bảng tổng hợp

| Vấn đề / Công cụ phát hiện | Mức độ | Nguyên nhân & Phân tích | Cách khắc phục |
| :--- | :--- | :--- | :--- |
| **Sử dụng PyTorch load không an toàn** (Bandit: B614) | Medium | Hàm `torch.load()` mặc định dùng `pickle` có thể bị khai thác thực thi mã độc nếu mô hình bị làm giả. | Đã thêm tham số `weights_only=True` vào hàm `torch.load()` trong `ai_service.py` để chỉ cho phép tải trọng số (weights) thay vì tải toàn bộ object, đảm bảo an toàn. |
| **Tải mô hình HuggingFace không an toàn** (Bandit: B615) | Medium | Gọi `from_pretrained()` mà không ghim revision cụ thể có thể dẫn đến việc tải mô hình bị thay đổi trái phép. | Lỗi này chủ yếu cảnh báo thực hành tốt. Vì đây là mô hình chuẩn `bert-base-uncased`, việc không ghim version ở mức độ dự án này là chấp nhận được. Đã dùng `# nosec B615` để tắt cảnh báo. |
| **Binding toàn bộ Interface** (Bandit: B104) | Low | Khởi chạy server FastAPI bằng `uvicorn.run` với `host="0.0.0.0"` làm server lắng nghe trên tất cả các mạng (kể cả mạng công cộng nếu port mở). | Đây là thiết lập cần thiết để Docker hoặc server CI/CD truy cập được ứng dụng. Đã đánh dấu `# nosec B104` trong `run.py`. |
| **Biến không sử dụng** (ESLint: `no-unused-vars`) | Error | Quên xóa các thành phần (hook, biến, import icon) sau khi refactor code, làm file bị rác. | Xóa bỏ các `import` thừa trong nhiều file (như `Navbar.jsx`, `Footer.jsx`). Xóa `setUser` trong các component Admin vì chỉ lấy user để đọc 1 lần. |
| **Gọi setState trong useEffect** (ESLint: `react-hooks/set-state-in-effect`) | Error | Gọi cập nhật state (như lấy data từ local storage, đóng menu) một cách đồng bộ trong useEffect sẽ kích hoạt chuỗi re-render (cascading renders). | 1. Đã chuyển logic khởi tạo giá trị đọc từ local storage vào thẳng hàm `useState(() => ...)` để xử lý đồng bộ và hiệu quả hơn.<br>2. Dùng comment `eslint-disable` cho các tình huống chủ ý (fetch data). |
| **Static Component lồng nhau** (ESLint: `react-hooks/static-components`) | Error | Khai báo hàm component con (`CustomTooltip`) ngay bên trong component cha (`AdminDashboard`) khiến con bị mount lại từ đầu mỗi lần cha render. | Kéo component `CustomTooltip` ra khỏi scope của component cha. |
| **Lỗi Hoisting** (ESLint: `react-hooks/immutability`) | Error | Định nghĩa hàm `const fetchLogs = ...` bên dưới `useEffect` - nơi gọi nó. Trong JS, từ khóa `const` không hỗ trợ đẩy khai báo lên trước (hoisting) như `function`. | Dời phần định nghĩa hàm `fetchLogs` lên ngay phía trước hook `useEffect` sử dụng nó trong `AdminLogs.jsx`. |
| **Ghi đè trường phân quyền (Authorization)** (React Doctor: `supabase-client-owned-authz-field`) | Critical | Trong trang Register và Login, Frontend đang cố ghi thêm các trường như `role: 'user'`, `status: 'Active'` và `id: userId` vào payload Insert. Điều này cho phép user thao túng quyền. | Gỡ bỏ hoàn toàn việc gọi `supabase.from('users').insert()` trên Frontend. Khuyến nghị thiết lập **Database Trigger** trên Supabase (khi có user mới ở auth.users thì tự động insert vào bảng users). |
| **Thiếu cấu hình Giảm chuyển động** (React Doctor: `require-reduced-motion`) | Warning | Framer-motion được sử dụng nhưng không có cấu hình tôn trọng cài đặt hệ thống của những người gặp vấn đề tiền đình (prefer reduced motion). | Bọc thẻ `<MotionConfig reducedMotion="user">` xung quanh toàn bộ Router tại `App.jsx`. |
| **Lỗi chạy Pytest Backend** (Test Framework) | Error | 1. Test runner gọi sai code.<br>2. Pytest không nhận diện được root dir.<br>3. Thiếu package `cv2` trên CI. | 1. Đổi code test qua TestClient.<br>2. Thêm `pythonpath = .` vào pytest.ini.<br>3. Thêm mock script cho ML/AI và `opencv-python-headless`. |
| **Logic Mật khẩu yếu** (Business Logic) | Lỗi Logic | Frontend trước đây không ép người dùng tuân thủ theo các quy tắc tạo mật khẩu mạnh. | Thiết kế lại `PasswordStrength.jsx`, cô lập luật mật khẩu vào `validators.js`. Ngăn chặn submit nếu mật khẩu không đạt. |
| **Giao diện không xử lý lỗi an toàn** (React Doctor) | Warning | Ứng dụng React có nguy cơ bị sập (white screen) nếu có lỗi xảy ra ở một component con mà không có Error Boundary bọc lại. | Đã tạo Component `ErrorBoundary.jsx` và bọc quanh thẻ `<App />` tại `main.jsx` để hiển thị UI lỗi thân thiện thay vì làm trắng trang. |
| **Hardcode thông tin Admin** (React Doctor) | Security / Warning | Code xử lý logic đăng nhập Admin bằng việc hardcode tài khoản và mật khẩu trực tiếp ở trang `Login.jsx`. | Đã gỡ bỏ block check hardcode admin, hiện tại toàn bộ ứng dụng chỉ sử dụng duy nhất Supabase Auth cho đăng nhập. |
| **Thiếu thông tin Supply Chain** (React Doctor: `low-supply-chain-score`) | Error | `package.json` thiếu các trường metadata như `description`, `author`, `license`, `repository` làm giảm điểm tin cậy của mã nguồn. | Đã thêm đầy đủ các trường `description`, `author`, `license`, và `repository` vào `package.json`. |
| **Lỗi Môi trường Test Frontend** (Vitest) | Error | Lỗi thiếu thư viện, JSX runtime crash do React 19 và dư file rác HTML. | Bổ sung vitest/jsdom, sửa lại Vite config, viết lại `App.test.jsx` thành unit test thuần cho logic và thêm `.gitignore` cho thư mục `coverage/`. |
